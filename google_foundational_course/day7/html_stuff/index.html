Representation

A great amount of time must be spent organizing the data and representing it correctly. For numeric features there is no problem since weights and biases are numerical, but for features that are strings we need another way of representing them. A solution might be one hot encoding, assigning a value to an element in a vector every time we run into that feature. We also need to pay attention to how the data is presented to us, if it is understandable, if there are any mistakes, if any number is out of range or if there are any magic values.

Cleaning Data

You might need to scale the data, if a feature has a far bigger range than another, the model will pay way more attention to the first one than the latter. The data also needs to be scrubbed, any values outside 3 stddevs may compromise your model.

Feature crosses

A feature cross is a synthetic feature that encodes non linearity. Since linear models can be scaled very easily, having linear features would be very helpful. You can cross two one hot vector that are not linearly related but their cross feature is. For example if we had: the feature x1 that could assume values a and b and x2 that could assume valuse c and d, then their cross feature, encoded in a 4D vector, tells us when we have a AND c or a AND d or b AND c or b AND d.
